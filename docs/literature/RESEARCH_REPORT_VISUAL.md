## 1. Arakawa 2012「TEM図の描き方の一例」 — デファクト標準

### 1.1 英文アブストラクトから抽出した **8ステップ**（※前調査の「7」は誤り）

1. インタビュー文字起こしと精読
2. セグメンテーション（分節化）
3. 各協力者のセグメントの時系列決定
4. 類似経験の位置づけとラベリング（"positioning similar experience and labeling"）
5. 分岐点（BFP）の決定
6. ポイント間の線描画と適切なラベル検討
7. 各インタビューイーのトレーサビリティ確認
8. 論文執筆

### 1.2 図1「TEM図」（p.97, basic example）から観察

- 長方形の並び（一直線的な径路）
- 右端に**二重線の長方形**（EFP = 等至点）
- 一部に**太線の長方形**と想定される形（OPP = 必須通過点）
- 径路を表す**黒の矢印**で連結
- 下辺に長い右向き矢印 + 「非可逆的時間」ラベル

### 1.3 図5「TEM図『ある人と親友になった過程』」（p.105, 完成例）

**今回の調査で最も重要な実例** — TEMの「お手本」。以下の要素を全て視覚確認:

| 要素 | 視覚的特徴 |
|---|---|
| **通常Box** | 白背景・黒細線の長方形、中央揃えテキスト |
| **EFP** | 右端の**二重線長方形** |
| **P-EFP** | EFPの上または下に**点線長方形** |
| **SD（社会的方向づけ）** | 図の**上部**に配置された**下向きの五角形**（ホームベース型を逆向きに）。複数個 |
| **SG（社会的ガイド）** | 図の**下部**に配置された**上向きの五角形**。複数個 |
| **径路の実線矢印** | 実線＋小さな黒塗り三角矢印 |
| **径路の点線矢印** | 点線＋小さな黒塗り三角矢印 |
| **非可逆的時間軸** | 図の最下部に大きな右向きの実線矢印 |

**重要な配置ルール**（今回の視覚確認で確定）:
- **SDは上から径路を「妨害」する向きで配置**（上→下の矢印）
- **SGは下から径路を「支援」する向きで配置**（下→上の矢印）
- SD/SGとも五角形のとがった側が径路に向く

### 1.4 補助的な図（分析プロセス図）

- 図2「それぞれの経験を時間順に並べよう」(p.101): 表形式（マトリクス）で、行=協力者、列=時系列セグメント
- 図3「同じような経験を同じ列にそろえて、ラベルを考えよう」(p.102): 同様の表＋列ラベリング
- 図4「分岐点の成り立ち」(p.103): 径路が分岐する様子を楕円＋矢印で示す

→ **TEMerPlus に「分析ワークシート」機能を追加する価値あり**（セグメント→カテゴリ化→図化の段階的UI）

## 2. Sato 2006「TEM原典」 — 提唱時の原型

### 2.1 図3「出会い関係に関する発達段階表示」(p.260)

- 横軸に「母物」「子ども」縦軸で時間
- シンプルな矩形・楕円の組み合わせ
- まだ「TEM図」として標準化された表記は確立されていない段階

### 2.2 図4「出会い関係に関するTEM」(p.260)

- 山なりに連なる形のTEM図（径路が上に伸びる表現）
- 右端に「等至点」のテキストラベル
- 「非可逆的時間→」の軸
- **Box自体の線種は細線の単純な矩形**（この時点ではEFP=二重線の作法はまだ確立していない）

### 2.3 図5「両極化した等至点」(p.264)

- **P-EFP概念の原図**
- 左から出発した径路が右側で**2つに分岐**
- 上側に「等至点」（EFP）、下側にも「等至点」（P-EFP）というテキストラベル
- **この時点ではP-EFPの明示的な形状規約はない**
- 下に「非可逆的時間→」

### 結論: 2006 → 2012 の進化

| 時期 | 表記 |
|---|---|
| 2006原典 | 単純な線・テキストラベル中心。EFP/P-EFPの形状区別なし |
| 2012 Arakawa以降 | **EFP=二重線**、**OPP=太線**、**P-EFP=点線**という区別が作法化 |
| 現代（カタログTEA 2023〜） | 色・領域塗りも併用するバリエーションが拡大 |

## 3. Toyota 2016「ビジネススクールTEA」 — 複雑な統合TEM図の実例

### 3.1 図1（p.348, 全面1ページの巨大TEM図）

非常に密度の高い**類型TEM図**（統合図）の好例:

- **上部に時期ラベル行**（テーブル形式）— 「入学前」「1年目」「2年目」「修了後」等の時期区分
- **左端に協力者ID列または変数列**
- **格子状の配置**で、時期×協力者のマトリクスに多数のBoxが整列
- 数十個のBoxが相互に矢印で連結
- **Box種別**: 角丸四角、六角形、二重線、通常矩形、楕円など多彩
- 多数の**SD/SG五角形**が上下に散在
- 図の右端に**巨大な右向き矢印**（非可逆的時間）
- 下部にキャプション「Figure 1. 専門職大学院ビジネススクール修了生の生涯学習型職業的アイデンティティ形成プロセス」

### 3.2 TEMerPlusへの示唆

- **時期ラベル列を上部に並べるグリッドレイアウト**が必要
- 統合TEM図では**数十〜100個のBox**を扱える必要あり
- Box種別は**最低でも5〜6種類**（通常、EFP、P-EFP、OPP、BFP、分岐楕円、注釈用）
- **自動整列機能**（時期軸ごとの縦列揃え）が不可欠

## 4. Kawai 2016「児童自立支援施設退所者」 — 凡例付き模範例

### 4.1 図1（p.5, 全面1ページTEM図）

**凡例（Legend）を図中右側に明示している模範的な例**。凡例から読み取れる記号セット:

| 記号 | 意味（確定したもの） |
|---|---|
| 実線枠の長方形 | 経験カテゴリー（実際に語られた） |
| **破線（点線）枠の長方形** | 想定される／潜在的な経験 |
| 角丸長方形 | カテゴリー別区分（別分類） |
| 楕円 | おそらくBFP相当 |
| 太線枠 | おそらくOPP |
| **下向き五角形（SD）** | 進路を逸脱させる社会的力 |
| **上向き五角形（SG）** | 進路を支援する社会的力 |
| 右端の大きな右向き矢印 | 非可逆的時間軸 |

### 4.2 径路表現の厳密性

- **実線径路**: 実際にインタビューで語られた径路
- **点線径路**: 想定可能だが実現しなかった径路
- この二分法が**非常に厳密に守られている**

### 4.3 TEMerPlusへの示唆

- **凡例の自動生成機能**は必須レベル（英語論文標準、日本語論文でも模範例で採用）
- 凡例の配置位置: **図の右側**または右下隅が多数派
- 凡例項目: 使用された記号のみ自動抽出して表示

## 5. Miyashita「TEM新たな理論的展開」 — TLMG/拡張表記

### 5.1 図1「ループモデル」(p.53, Zittoun & Gillespie 2016)

- 円環状のループ図
- ルーフモデル・一般化・トロポス（tropos）・時間軸のラベル
- **標準的TEM図ではない**、理論拡張モデル

### 5.2 図2「記号論におけるTEMとイマジネーションモデル」(p.57, 著者作成)

- **TEMを基盤に拡張した独自表記**
- 横軸に OPP → BFP → EFP という時系列的な標準配置
- **「イマジネーション」**というラベルの吹き出し的な曲線要素が上部
- 「記号圏」「過去の時間」「可能性 non-A」等の領域ラベル
- 標準TEM図に**時間軸拡張・可能性領域を重ね描き**

### 5.3 図4・図5「TEM研究の類型」(p.59)

- マトリクス（2軸）による研究類型化
- 人型アイコンの使用（対象者表現）
- 「類型」「規範」「P-Non-B」等のラベル

### 5.4 TEMerPlusへの示唆

- TLMGを含む**拡張表記は標準化されていない**（Miyashita は著者独自）
- **基本TEM図に上書き（レイヤー）する機能**が将来的に必要
- 人型アイコン等の**シンボルライブラリ**（Nice to have）

## 6. 総合的な表記ルール（視覚確認の結果）

### 6.1 厳格なルール（全例で遵守）

- **径路の実線＝実現、点線＝未実現**
- **EFPは右端（時間軸終端）に配置**
- **非可逆的時間の矢印を図に明示**
- **SD/SGは五角形で、向きで区別**

### 6.2 作法的ルール（2012以降の大多数で遵守）

- **EFP=二重線枠**
- **P-EFP=点線枠**（EFPの対として配置）
- **OPP=太線枠**
- **BFP=実線楕円 または 実線長方形**
- 通常Box=細線長方形

### 6.3 揺れ・独自性（論文ごとに異なる）

- **凡例の有無と配置**（右側 / 下部 / なし）
- **Box内テキストの配置**（中央 / 左寄せ）
- **色の使用**（モノクロが多数だが、カラー化が進行中）
- **時期ラベルの位置**（図の上部 / 下部 / 時間軸上）
- **協力者ID/ラベルの表示有無**
- **BFPの形状**（長方形 / 楕円 / 六角形）

## 7. TEMerPlus実装への具体フィードバック

### 7.1 **確定された必須要素**（視覚確認で裏付け）

- 通常Box（細線長方形）
- EFP（**二重線**長方形）
- P-EFP（**点線**長方形）
- OPP（**太線**長方形）
- BFP（**実線**長方形 または **楕円**）
- SD五角形（**下向き**、図の**上部**配置）
- SG五角形（**上向き**、図の**下部**配置）
- 実線径路矢印
- 点線径路矢印
- 非可逆的時間軸（図端の長い矢印）

### 7.2 **現TEMerPlus実装の要検証ポイント**

前回レポートで「EFPとOPPの線種が逆転している可能性」を指摘したが、**視覚確認で確定**:
- **EFP=二重線** ← 文献標準
- **OPP=太線** ← 文献標準
- 現TEMerPlus (`Module_Make_Box.bas`) の `ApplyLineStyles` 実装を要確認

### 7.3 **追加推奨機能**（視覚確認で再確認）

1. **凡例の自動生成**（★★★）: Kawai 2016 模範例、英語論文で標準
2. **時期ラベル列**（★★★）: Toyota 2016 のような統合図で必須
3. **統合/類型TEM図モード**（★★★）: Toyota 2016 のような複数協力者を束ねる機能
4. **分析ワークシート**（★★）: Arakawa 2012 の図2-4 のような段階的UI（セグメント表→カテゴリ化→図化）
5. **BFPに楕円形オプション**（★★）: Kawai 2016 で採用
6. **協力者ID列管理**（★★）: 統合図で必須
7. **凡例の配置自動化**（★）: 右側 / 下部選択可能

### 7.4 **保留・慎重検討**

- TLMGの3層スイムレーン: 標準表記がなく、著者ごとに独自（Miyashita も独自）→ **汎用テンプレート提供よりカスタムレイヤー機能**が適切
- 色使い: 現時点でモノクロ主流。カラー対応は将来検討

## 10. 結論

**現TEMerPlus実装は文献上の必須要素をほぼ網羅**。ただし:

- ✅ **視覚確認により線種マッピングの文献標準が確定**: EFP=二重、P-EFP=点線、OPP=太線、BFP=実線
- ⚠️ **現実装がこのマッピングと整合しているか要検証**（`old_file/VBA_Backup/Module_Make_Box.bas`の `ApplyLineStyles`）
- ➕ **追加価値の大きい機能**: 凡例自動生成、時期ラベル列、統合TEM図モード、分析ワークシート
- 🎯 **戦略的位置**: 視覚確認でも「TEM専用作図ツールは存在しない」が補強された。TEMerPlus が**事実上の標準**を提示する立場
