# テスト実施ガイド

## 0. 事前準備

```bash
cd webApp
npm install         # 初回のみ
npm run dev         # 開発サーバ起動
```

ブラウザで `http://localhost:5173/` を開く。

## 1. 基本動作の素早い確認（5 分）

1. 初期サンプル図が表示される
2. Box をドラッグで動かす
3. Box をダブルクリックでラベル編集
4. Ctrl+S で保存（Chrome/Edge はファイル保存ダイアログ）

## 2. CSV インポートの動作確認

`test-assets/samples/` 以下のファイルを使用。

### 2-1 最小（ラベルだけ）
1. リボン「ファイル > CSV インポート…」
2. `labels-only.csv` を選択
3. 自動判定（ヘッダなし、1 列のみ → label）を確認
4. **「現在のシート末尾に追加」** で実行
5. 5 個の Box が等間隔で右へ追加されることを確認

### 2-2 種別列あり（英名）
1. 新規シート作成
2. `with-types.csv` を選択
3. ヘッダ自動判定を確認
4. 2 列目が「種別」に自動割当されていることを確認
5. インポート → 各 Box が適切な種別で作成される
6. **順次接続 ON** を試す → RLine が自動で追加される

### 2-3 種別列あり（日本語）
1. 新規シートで `japanese-types.csv`
2. 「分岐点 → BFP」「等至点 → EFP」等が自動マップ
3. インポート後、凡例の自動生成が反映される

### 2-4 全列指定
1. `full-columns.csv`（label/type/timeLevel/itemLevel/subLabel/description）
2. 各値が正しく反映されることを確認
3. 指定した Time/Item Level で配置されている

### 2-5 Box 間挿入
1. 既存シートの Box1 直後に挿入したい
2. 「指定 Box の間に挿入」を選ぶ
3. 開始 Box に Box1 を選択
4. インポート → Box1 の右に新規が入り、以降の Box が右シフト

## 3. エクスポートの動作確認

### 3-1 PNG / SVG
1. ファイル「出力…」
2. プレビュー左で プリセット「論文用 A4」
3. 背景「白」、要素オプションは全 OFF
4. PNG で保存 → 生成された画像を確認

### 3-2 PDF
1. フォーマット「PDF」
2. 用紙サイズ A4、マージン 0.3
3. 保存 → PDF を開き、画像がページ内に収まっていることを確認

### 3-3 PPTX（要実機確認）
1. フォーマット「PPTX」
2. 用紙サイズ 16:9、スケーリング ON
3. 保存 → PowerPoint で開く
4. **二重線**（EFP）が細くて 2 本線になっているか確認（DrawingML の `cmpd="dbl"`）
5. **SDSG がブロック矢印**（五角形ではなく）で向きが正しいか
6. **レイアウトに応じた用紙方向**（横型 → 横スライド）

## 4. 論文レポートの動作確認

1. ファイル「論文レポート…」
2. 概要タブ: タイトル・著者入力
3. 記号体系タブ: 「荒川・安田・サトウ 2012」選択
4. 協力者タブ: 人数・記述入力
5. インタビュータブ: 方法・期間入力
6. 表記方針タブ: 「横軸の長さ」にチェック + 説明入力
7. 出力タブ: 「現在のキャンバスを図として含める」ON
8. 「レポートを生成」→ `.docx` を Word で開く
9. タイトル / 各節 / 図 / 要素一覧表が整形されて出ている

## 5. 印刷レイアウト調整（ExportPreviewDialog）

1. ファイル「出力…」
2. プレビューを見ながら
3. フィット「横だけ収める」を試す（縦ははみ出し許容）
4. 「両方に収める」で全体が用紙内に
5. 文字サイズ ± +2、線太さ ± +0.5 で強調
6. **元のキャンバスは変わっていないこと**を確認（ダイアログを閉じて戻る）

## 6. シート全体のリサイズ・一括移動

### 6-1 リサイズ
1. 編集リボン「シート > リサイズ…」
2. 「用紙サイズにフィット > A4 横 > 両方に収める」
3. 倍率プレビューを確認
4. 適用 → シート全体がリサイズ
5. Ctrl+Z で戻る

### 6-2 一括移動
1. 編集リボン「シート > 一括移動…」
2. Time_Level 方向に +2、Item_Level 方向に +1
3. 適用 → Box と SDSG だけが移動、時期区分・時間矢印は不変

## 7. プロパティパネルと装飾

1. Box を選択
2. PropertyPanel のタブ「基本 / 装飾・調整 / 論文」を切替
3. 基本タブ: ラベル入力欄の装飾ツールバーを使う
   - 「重要」を選択 → B ボタン → `<b>重要</b>`
   - 色ボタン → `#cc0000` → 赤に
4. 装飾・調整タブ:
   - 「文字を Box に合わせる」
   - 「Box を文字に合わせる」
   - autoFitText ON で自動調整
5. 論文タブ:
   - 説明文を入力 → 論文レポートに反映されることを確認

## 8. スマートガイド・アライン

1. 2 つの Box を作成
2. 片方をドラッグして近づける
3. 端（左・右・中央・上・下・中央）で揃うと**ピンクの点線**
4. 複数選択 + 整列リボン「左揃え」などでまとめて揃う
5. 「幅揃え」「サイズ揃え」（複数選択時）

## 9. 凡例編集

1. キャンバス上の凡例を**クリック**（シングル）
2. PropertyPanel に凡例全設定が表示される（タブ構造）
3. タイトル位置: 上 → 左 に切替
4. 列数: 横型 2 列に
5. 項目別: 「EFP」のラベル上書き → 「到達点」に変更
6. 「2 行目非表示」トグル
7. 凡例を**ダブルクリック**でも設定ダイアログが開く

## 10. 用紙枠設定

1. リボン表示タブで「用紙枠」ON
2. A4 2 枚、黒枠、枠外薄グレーが表示される
3. レイアウト切替で長辺方向が自動回転
4. ファイル > 設定 > 全体タブ > 用紙枠
5. A3 に変更、枚数 3 に変更 → 即反映
6. 色を赤に → 枠線が赤に

## 11. React Flow コントロール

1. 右下のボタン群
2. ＋ / − で Zoom
3. ⛶ で全体 fit（時間矢印・時期区分・凡例を含む全要素）
4. ↔ で横幅 fit
5. ↕ で縦幅 fit
6. **縦型レイアウトで横 fit** → 時期ラベルも収まる
7. 🔓 → 🔒 で編集ロック

## 問題が出たら

- 再現手順をメモ（何を選択、どのボタン）
- ブラウザ / OS / 解像度を記録
- コンソールエラーのスクリーンショット（F12）
- `.tem` ファイルが再現に使える場合は添付

```
（どう見えているか）

## 再現手順

2. ...
3. ...

## 期待
（こうあるべき）

## 実際
（こうなっている）

## 環境
- ブラウザ:
- OS:
- 画面サイズ:

## 添付
- スクリーンショット
- .tem ファイル（問題を再現できるもの）
```
