# TEMer 機能仕様書

バージョン: 0.1-draft
最終更新: 2026-04-18

TEM（複線径路等至性モデル）作図支援ツール「TEMer」Web版の機能仕様。

---

## 0. エラー・警告の扱い

- **基本**: 画面下からのトースト通知（数秒で自動消去）
- **致命的エラー**: モーダルダイアログ（ユーザ操作を要求）
- **軽微な警告**: インライン表示（該当フィールドの横）
- **通知カテゴリ**: info（青）/ success（緑）/ warning（黄）/ error（赤）

## 0.1 視覚デザインの基本

- **配色**: **標準はモノクロ**（文献の主流）、多色テーマも用意
- **フォント**: システムフォント（OS依存の最適化）
  - 日本語: Noto Sans JP / Hiragino / Yu Gothic を自動選択
  - 英数字: system-ui / -apple-system / Segoe UI
- **Box 既定サイズ・グリッド間隔・スナップ閾値**等の細部数値は実装しながら調整

## 1. 全体方針

### 1.1 コンセプト
- **世界初のTEM専用作図ツール**を目指す（文献調査で競合不在を確認）
- **文献標準（Arakawa 2012ら）をデフォルト提供**するが、**画一化は強制しない**
  - サトウ先生（TEM創始者）が標準化に反対の立場のため
  - TEMerPlus は「標準を強制する」のではなく「**研究者が自分の表記選択を明文化するのを支援する**」ツール
  - カスタマイズは完全に自由
- **Figma/PowerPoint風UX**で、学習コストを最小化
- **データドリブン**: 図形はデータから生成され、データは図形編集で更新される

### 1.2 対象ユーザー
- **第一ターゲット**: 看護学・教育学・発達心理学・キャリア研究等でTEMを使う研究者
- **第二ターゲット**: 大学院生（学位論文）、学部生（卒論）
- **将来ターゲット**: 英語圏研究者（Springer 2026刊行書籍に合わせた国際展開）

### 1.3 プラットフォーム
- **Webアプリ（ブラウザベース）** を第一形態
- 推奨ブラウザ: Chrome / Edge（File System Access API対応）
- Safari / Firefox でも基本機能は動作
- 将来的にTauriでデスクトップ化も視野

### 2.2 リボンUI（上部）

PowerPoint/Officeに近い構造。7つのタブで整理:

#### Home タブ
- クリップボード: 切取/コピー/貼付/書式コピー
- Undo/Redo
- フォント/装飾: サイズ・太字・斜体・下線・色
- 整列: 左揃え/中央/右揃え、等間隔、上下配置

#### Insert タブ
- **Box挿入**（ドロップダウンで通常/OPP/BFP/EFP/P-EFP/注釈）
- **Line挿入**（ドロップダウンで実線/点線）
- **SD** / **SG** ボタン
- 吹き出し注釈
- 時期ラベル
- 凡例
- **画像挿入**（ロゴ等）
- テキストボックス

#### Design タブ
- レイアウト方向（縦型/横型切替）
- テーマ/配色
- **テンプレート一覧**
- 用紙サイズ（A4/A3/16:9/4:3/カスタム）
- 用紙枠の表示/非表示
- 非可逆的時間矢印のON/OFF

#### View タブ
- ズーム（%/フィット/幅に合わせる）
- グリッド表示ON/OFF
- スナップON/OFF
- **データシート表示/非表示**
- **プロパティパネル表示/非表示**
- 凡例表示
- コメント表示/非表示

#### Share タブ
- エクスポート: PNG/SVG/PPTX/PDF
- **横分割エクスポート**
- **論文用レポート出力**（Word+pptx）
- 印刷

#### File タブ
- 新規 / 開く / 保存 / 名前を付けて保存
- 最近使ったファイル
- `.tem` インポート/エクスポート
- 旧 `.xlsm` マイグレーション

#### Help タブ
- ウォークスルー再実行
- マニュアル
- 動画チュートリアル
- バージョン情報

### 2.3 中央: キャンバス

- React Flow ベース
- ドラッグ、リサイズ、グリッドスナップ、自動整列、複数選択、ズーム・パン
- 背景に**非可逆的時間軸**（右向き矢印）を自動描画
- **用紙枠**（ONの場合）点線で表示、はみ出し部分を警告色で強調

### 2.4 左: データシート（折りたたみ式）

- **クリック1回で最小化⇔展開**（サイドバー最小化時はアイコンのみ表示、🗂 マークで識別）
- 用途: **初期一括作成 + 複数Boxコピペによる追加**
- タブでBox/Line/SD/SG/Commentsを切替（データシート内の内部タブ）
- ソート、フィルタ、セル一括編集対応
- **シート単位**（現在のシートタブのデータのみ表示）
- 日常編集はここを使わない — 右プロパティパネル優先

#### 最低限の列構成

**Box タブ:**
| 列 | 必須 | 備考 |
|---|---|---|
| ID | ✓（自動） | 自動採番 |
| 種別 | ✓ | 通常/BFP/EFP/P-EFP/OPP/注釈 |
| ラベル | ✓ | Box内テキスト |
| Item_Level | ✓ | 縦軸位置 |
| Time_Level | ✓ | 横軸位置 |

**Line タブ:**
| 列 | 必須 | 備考 |
|---|---|---|
| ID | ✓（自動） | 自動採番 |
| 種別 | ✓ | 実線 or 点線 |
| From | ✓ | 始点Box ID |
| To | ✓ | 終点Box ID |

**SD/SG タブ:**
| 列 | 必須 | 備考 |
|---|---|---|
| ID | ✓（自動） | |
| 種別 | ✓ | SD or SG |
| ラベル | | 説明文 |
| 対象 | ✓ | 紐付ける Box/Line ID |

その他の詳細プロパティ（サイズ、フォント、接続モード等）は**プロパティパネルで編集**。データシートは**最小限に留める**方針。

### 2.5 右: プロパティパネル（Figma風）

- **クリック1回で最小化⇔展開**（最小化時はアイコンのみ表示、▼ マーク）
- 図形選択時に即座に表示
- **単一選択**: すべてのプロパティを個別編集
- **複数選択**: 共通プロパティを一括編集、異なる値は「混在」表示
- **無選択**: シート全体の設定（時間軸、凡例、用紙サイズ等）

### 2.6 下部: シートタブ（Excel風）

**複数人のTEM図を1つの `.tem` ファイル内で管理**。タブで切替。

#### 各シートの独立性
- 各シートは**独立したTEM図**
- Box/Line/SD/SG/コメントは各シートに属する
- **協力者1人 = 1シート**、**統合TEM図も1シート**としてまとめる運用

#### タブ操作
- 左右クリック: シート切替
- 右クリック: 名前変更 / 複製 / 削除 / 並び替え / シート間コピー
- ダブルクリック: 名前変更
- `+` ボタン: 新規シート追加
- ドラッグで順序変更

#### 共有要素（全シート共通）
- 設定（レイアウト方向、フォント、用紙サイズ等）
- 凡例定義（統一された記号体系）
- メタデータ（協力者情報、インタビュー方法等）

#### シート間の Box/Line コピー

- 選択した Box/Line/SD/SG を **他のシートにコピペ可能**（Ctrl+C / Ctrl+V、またはメニュー）
- ペースト時に ID は新規採番
- 統合TEM図シートは**手動作成**（各シートから Box をコピペして集約）
- **将来Phase**: 統合TEM図の自動生成機能（Phase 6）

#### 複数選択・コピー・ペースト
- 矩形選択または Shift+クリックで複数選択
- Ctrl+C: 選択要素をクリップボードに
- Ctrl+V: 相対位置を保ってペースト（少しオフセット）
- Ctrl+D: 選択要素を複製（コピー+ペースト）

#### シート比較（Phase 6）
- **「分割表示」**: 2つのシートを左右に並べて同時表示

### 2.6.1 マグネット吸着・整列機能

ドラッグ時の自動整列機能。PowerPoint方式のソフトスナップ。

#### 3種のスナップ（個別ON/OFF可）
1. **整列ガイド**: 他のBoxと**左端/中央/右端/上端/中央/下端**が揃う位置で吸着 + 赤/青の点線ガイドライン表示
2. **距離スナップ**: 他のBoxから**一定距離（既定20px）**の位置で吸着（Box同士が等間隔に並ぶ）
3. **グリッドスナップ**: 背景グリッドに吸着（既定10px）

#### 動作
- **通常時は全てON**
- **リボン View タブにON/OFFトグル**配置
- **Alt キー押下中は一時無効化**（自由配置）
- 設定画面でスナップ強度・距離調整可

### 2.6.2 ラベル編集（2方式併用）

#### A. プロパティパネル入力
- Box/Line をクリック → 右のプロパティパネルに「ラベル」フィールド
- テキストエリアで編集、エンターまたはフォーカス外れで確定

#### B. ダブルクリックでインライン編集
- Box をダブルクリック → その場でテキスト編集モード
- Enter または外側クリックで確定
- Esc でキャンセル

### 2.6.3 図形の重なり順（Z-order）

基本は重ならない設計だが、以下のユースケースで調整可:
- 注釈 Box を他Boxの上に重ねる
- 吹き出しラベルを径路の上に配置
- 統合TEM図で径路が交差する時の前後関係
- 背景画像（ロゴ等）を最背面に

#### 操作方法
- **右クリック → 前面へ / 背面へ / 最前面 / 最背面**
- **Home リボンにボタン**
- **ショートカット**:
  - `]`: 1つ前面へ
  - `[`: 1つ背面へ
  - `Shift+]`: 最前面
  - `Shift+[`: 最背面

### 2.7 下部: ステータスバー

- ズーム率
- グリッド状態
- スナップ状態
- 用紙サイズ
- 選択数
- 保存状態（未保存なら警告）

## 3. TEM図の構成要素（記号仕様）

### 3.0 新規Box の配置モード（3種併用、デフォルト=自動配置）

新規 Box を追加する際の配置方法を3種サポート:

| モード | 説明 | トリガー | 既定 |
|---|---|---|---|
| **自動配置（D）** | 選択中BoxのTime_Level+1 に自動配置 | リボンの「Box追加」ボタン押下 | ✅ |
| **隣接配置（B）** | 最後に選択した Box の右隣に自動配置 | 「隣接追加」ボタン or キーボードショートカット | |
| **クリック配置（C）** | Insert → Box選択 後、キャンバスでクリックした位置に配置 | Insert ドロップダウンから Box選択 | |

- 設定でデフォルトを変更可能
- キーボードショートカット:
  - `Ctrl+Shift+N`: 自動配置（D）
  - `Enter`: 隣接配置（選択Boxがあるとき）

### 3.1 Box 種別

| Type | 枠線 | 太さ | 既定形状 | 意味 |
|---|---|---|---|---|
| normal | solid | 1.5pt | 長方形 | 通常の経験ノード |
| BFP | solid | 1.5pt | 長方形（楕円選択可） | 分岐点 |
| **EFP** | **double** | **2.5pt** | 長方形 | **等至点（二重線）** |
| P-EFP | dashed | 1.5pt | 長方形 | 両極化等至点 |
| **OPP** | solid | **3.0pt** | 長方形 | **必須通過点（太線）** |
| annotation | dotted | 1.0pt | 長方形 | 注釈・補足 |

将来追加: 2nd EFP, P-2nd EFP, 促進的記号（Promoter Sign）

### 3.2 Line 種別

| Type | 線種 | 矢頭 | 意味 |
|---|---|---|---|
| RLine | solid | 三角 | 実現した径路 |
| XLine | dashed | 三角 | 想定された径路（未実現） |

- **デフォルトは直線**
- **曲線オプション**: Lineプロパティで `straight` / `curve` を切替可能（曲線は bezier）
  - 文献上は直線が圧倒的多数だが、TLMG や三層モデルの変容表現で曲線が使われる例がある（Miyashita 論文等）
- 太さ: 1.5pt 既定

#### 接続モード（2種類、Line ごとに選択可）

| モード | 説明 | 既定 |
|---|---|---|
| **center-to-center** | 始点Boxの右辺中点 → 終点Boxの左辺中点（斜めになり得る） | ✅ |
| **horizontal** | 始点Boxの右辺の任意Y → 終点Boxの左辺の同一Y で水平接続 | |

- デフォルトは `center-to-center`
- Lineプロパティパネルで切替可能
- 一括変更: 複数Line選択→まとめて変更
- 文献上は両方が使われている（Kawai 2016 は center-to-center 主流、Toyota 2016 は混在）

### 3.3 SD / SG（五角形）

| Type | 形 | 向き | 配置 |
|---|---|---|---|
| SD | 下向き五角形 | 径路を上から妨害 | 図の上部 |
| SG | 上向き五角形 | 径路を下から支援 | 図の下部 |

### 3.4 時間軸

- **非可逆的時間矢印**を**自動挿入**（図の下端または端）
- 右向き矢印 + 「非可逆的時間」ラベル
- 縦型では下向き矢印 + 右端
- 時期ラベル（例: 「入学前」「就職後」）を時間軸上に離散配置可能

### 3.5 コメント機能（Figma風）

Box/Line にピン止めされた注釈メモ。

#### 追加方法（両方サポート）
- **A**: Box/Line を**右クリック → コメント追加**
- **B**: リボンの「コメントモード」ON → 任意の Box/Line クリックでコメント追加

#### UI
- コメントが付いた要素に**💬 アイコン**がオーバーレイ表示
- クリックでポップアップ、スレッド形式で返信可能
- 解決/未解決ステータス
- 右サイドバーで全コメント一覧表示可能
- View メニューで「コメント表示/非表示」切替

### 3.6 凡例（Legend）

- **自動生成**: 実際に図中で使用された記号のみ抽出
- 配置: 右側 / 下部 / 外部ページ から選択
- 日英切替対応

```typescript

      horizontalLength?: { hasMeaning: boolean; description?: string };
      arrowAngle?: { hasMeaning: boolean; description?: string };
      verticalPosition?: { hasMeaning: boolean; description?: string };

  };

  type: 'individual' | 'integrated'; // 個別 or 統合

  y: number;
  width: number;
  height: number;
```

## 6. エクスポート仕様

### 6.0 論文報告エクスポート（新機能）

TEM図と合わせて**研究報告用のテキスト資料**を生成。論文Methods/Results章にそのまま使える情報を出力。

#### フォーマット
- **Word (.docx)**: 編集可能な報告書
- **pptx 付録スライド**: 本編の後に続くサマリーページ

#### 設計哲学
- **強制せず、リマインドのみ**（サトウ先生の画一化反対の立場を尊重）
- ユーザが入力した範囲でのみ出力（未入力項目はスキップ）
- 「説明不要（自明）」チェックで個別要素を例外化可能
- 論文用エクスポート時のみ、未入力項目を柔らかくリマインド（強制ではない）

#### 出力内容（ユーザ入力があるもののみ）

1. **図のメタデータ**
   - タイトル、作成日、作成者、概要
   
2. **記号体系の宣言**
   - 採用した記号体系（Arakawa 2012 準拠 / 独自拡張 / カスタム）
   - カスタム時は具体的定義

3. **協力者情報**
   - 人数、属性、サンプリング方法
   - HSI（Historically Structured Inviting）の論理
   - 仮名・ID リスト

4. **インタビューの方法**
   - 方法（半構造化、非構造化等）
   - 期間・回数
   - 分析手法（KJ法・M-GTA 併用等）

5. **表記方針の明文化**（任意、ユーザが意味を付与した場合のみ）
   - 横軸（Time_Level）間隔: 「実時間に比例」「等間隔・便宜上」「密度を反映」等
   - 矢印の角度: 「傾きが変化の急峻さを表す」「視認性のため」等
   - 縦軸（Item_Level）位置: 「価値・感情の高低」「視認性のため並列」等
   - 色の使用: 「協力者識別」「類型識別」「使用せず」等
   - 線の太さ: 「主要径路強調」「標準通り」等

6. **要素一覧**
   - Box 一覧（ID・種別・ラベル・説明）
   - Line 一覧（from→to・種別・説明）
   - SD/SG 一覧（対象・説明）
   - 時期区分（時期ラベル・範囲・説明）

#### UI（論文用エクスポート時のチェック）

```
[論文用エクスポート前の確認]

入力状況:
  ✓ 協力者情報: 入力済み
  ✓ インタビュー方法: 入力済み
  ⚠ 横軸の意味: 未入力（リマインド）
  ⚠ Box説明: 8/10 件入力済み（2件リマインド）

  [ ] このまま出力する
  [ ] 記入を続ける
```

### 6.0.1 エクスポート時の自動縮尺調整（重要な設計方針）

**編集中は用紙サイズを意識せず自由に作図。エクスポート時に自動調整する**ことで、研究者は「用紙に収まるか？」を考える負担から解放される。

#### 動作
1. エクスポート設定で**用紙サイズ**を指定（A4/A3/16:9/4:3/カスタム）
2. 図の**バウンディングボックス**を計算
3. 用紙サイズに**プロポーショナルに縮尺調整**して収める
4. **矢印の長さ・Box のサイズ**も自動スケール
5. 文字サイズは可読性を保つため、一定以上は縮小しない（下限あり）

#### ユーザ選択オプション
- **A. 自動フィット**: 全体を用紙にピッタリ収める（デフォルト）
- **B. 自動分割**: 用紙に収まらない場合、**横分割エクスポート**を使う
- **C. 等倍+切り取り**: 縮小せず、用紙サイズでクロップ
- **D. プレビュー → 手動調整**: 自動結果を確認して、分割位置等を手動で調整

#### 下限設定
- 最小文字サイズ: 8pt（可読性の限界）
- 最小Box幅: 30px
- 制約を超えて縮小できない場合は、自動的に「分割」を提案

### 6.1 PPTX
- **ライブラリ**: PptxGenJS
- スライドサイズ: 16:9 (13.33 x 7.5 inch) 既定、設定で変更可
- **Box**: `addShape` (rect / ellipse) + `addText`
- **二重線（EFP）**: 2矩形重ね描き（PptxGenJSが `cmpd="dbl"` 未対応のため）
- **点線（P-EFP, XLine）**: `dashType: 'dash'` or `'dashDot'`
- **太線（OPP）**: `width: 3.0`
- **矢印**: `addShape(line, { endArrowType: 'triangle' })`
- **凡例**: 凡例用矩形グループを右下に自動配置（ON時）

#### 横分割エクスポート

| 項目 | 仕様 |
|---|---|
| **用途** | 横に長いTEM図を pptx スライド幅に収めるため、**任意位置で N 分割** |
| **フォーマット** | **単一pptxに複数スライド**（slide 1, 2, 3...） |
| **分割位置指定** | 分割数指定（2/3/4...）で均等配置、または縦線ドラッグで手動調整 |
| **オーバーラップ** | **ユーザ設定可能**（デフォルト 50px）。隣接スライドの境界部分を重複表示 |
| **Boxの扱い** | 分割線をまたぐBoxは**両方のスライドに複製表示** |
| **矢印の扱い** | 分割線をまたぐ矢印は、左スライド右端に「**→続**」、右スライド左端に「**続→**」の続きマーカー表示。矢頭は本来の終点側のみ |
| **凡例** | **全スライドに表示** |
| **時間軸矢印** | **各スライドに部分的に描画**（全体で1つの時間軸を構成、左端から続くように） |
| **時期ラベル** | 各スライドで該当時期のラベルのみ表示 |
| **SD/SG** | 親図形（Box/Line）が表示されるスライドに従って表示 |
| **UI** | エクスポート前にプレビュー画面で分割線位置・オーバーラップ量を調整可能 |

**実装イメージ:**
```
[Preview Panel]
┌─────────┬─────────┬─────────┐
│Slide 1  ┆Slide 2  ┆Slide 3  │
│         ┆         ┆         │
│[Box1]→[┆Box2]→[OP┆P]═[EFP]│  ← 分割線をBoxがまたぐ
│         ┆         ┆         │
│     →続┆続→     ┆         │  ← 矢印の続きマーカー
│[非可逆的時間→]───────────→ │  ← 時間軸は全スライド通貫
└─────────┴─────────┴─────────┘
[Split count: 3▼] [Overlap: 50px] [Export]
```

## 6.5 論文執筆支援機能（調査発見機能）

TEMerPlus を他ツールから差別化する独自機能群。優先度はPhase 3以降。

### 6.5.0 縦書き対応（Phase 2で実装、必須）

Box テキストを縦書きにする機能。研究者の強い要望あり。

**機能:**
- Box プロパティで `textOrientation: 'vertical'` を選択
- **長音記号「ー」が縦向きに正しく表示される**（重要）
- 数字・英字は通常通り横向き（日本語縦書き慣習に従う）
- Box のサイズも縦長方向に自動調整（オプション）
- pptx/SVG エクスポート時も縦書きが保持される

**CSS実装:**
- `writing-mode: vertical-rl` (縦書き右→左、日本語標準)
- `text-orientation: mixed` （英数字は横向き、日本語は縦向き）

### 6.5.1 本文 ↔ 図中ラベルの自動同期（Phase 3、優先度中）

論文のMethods/Results章に書いたカテゴリー名と、図中のBoxラベルの不一致を検出・同期。

**機能:**
- 論文テキスト（Word/Markdown/プレーン）を取り込み
- 図中のBoxラベルと本文のカテゴリー名を照合
- 不一致を**リストで警告表示**
- 一括修正・個別修正の選択可

**ユースケース:**
- 投稿前の最終チェック
- 査読後の修正時のラベル整合性確認

### 6.5.2 逐語録メンション（Phase 4、優先度低）

**設計課題**: 逐語録のフォーマットが定義されていないため、先にフォーマット設計が必要。

Box ごとに「このカテゴリーはどの発話から抽出されたか」をリンク管理。

**機能:**
- Box プロパティパネルから**逐語録テキスト**を貼り付け/ファイル読み込み
- Box と逐語録の**特定箇所を紐付け**
- 論文用エクスポート時に「各Boxの根拠発話」として付録に自動出力
- 複数発話の紐付け可

**ユースケース:**
- 査読コメント対応時の根拠提示
- 分析のトレーサビリティ確保
- 共同研究者とのレビュー時の議論根拠

### 6.5.3 スペルチェック（優先度低）

ブラウザネイティブのスペルチェック（日英）を Box/Line/SD/SGラベルに適用。

- `spellcheck="true"` をテキスト入力フィールドに設定
- 日本語: 利用可能なエンジンがあれば（Chromeは一部対応）
- 英語: ブラウザ標準で対応

### 6.5.4 LLM API連携（Phase 5 以降、優先度低）

ユーザのAPIキーを使って、**作図支援**を LLM で補助:

#### 想定機能
- **ラベル候補の提案**: Box 内の短縮ラベルから、より適切な表現を提案
- **分岐点・等至点の自動提案**: 逐語録をペーストすると、BFP/EFP 候補を抽出
- **凡例説明文の生成**: 記号体系の説明を自動下書き
- **論文 Methods 節のドラフト作成**: メタデータ + 図情報から自動生成
- **図の構造レビュー**: 論理的整合性のチェック（EFP/P-EFPの対応、OPP の配置）

#### APIキー管理
- 設定画面でユーザが入力（Claude / OpenAI / Gemini 選択可）
- **キーはユーザのブラウザローカルにのみ保存**（サーバ送信しない）
- 利用は完全オプトイン

#### 注意
- **TEMerPlus本体は LLM なしで完全動作**。LLM は補助機能のみ
- 優先度は低い（Phase 5 以降）

## 6.6 保存と自動バックアップ

### 6.6.1 正式な保存（`.tem` ファイル）
- **明示保存のみ**（Ctrl+S）
- 初回保存は「名前を付けて保存」ダイアログ
- 以降は同じファイルに上書き
- File System Access API（Chrome/Edge）: ネイティブ並みの上書き保存
- Safari/Firefox: ダウンロードフォルダに保存される

### 6.6.2 自動バックアップ（IndexedDB）
- **30秒ごと** に現在の状態を IndexedDB に保存（サイレント、ユーザに意識させない）
- ブラウザクラッシュ・タブ誤閉じ・PC シャットダウン等の事故対策
- **次回起動時**: 未保存のデータが検出されたら**復元ダイアログ**を表示
  ```
  前回保存されていないTEM図が残っています。
    [復元する]  [破棄する]  [一旦保留]
  ```
- PowerPoint の「自動回復」機能と同等

### 6.6.3 ファイル内部構造

```json
{
  "version": "0.3",
  "sheets": [ /* 各シート */ ],
  "metadata": { /* 全体メタデータ */ },
  "resources": {
    "images": {
      "img_001": "data:image/png;base64,iVBOR..."
    },
    "transcripts": {
      "tsc_001": "インタビュー逐語録のテキスト..."
    }
  }
}
```

- **外部リソース（画像・逐語録）は全て内部埋め込み**（Base64 or plain text）
- ファイル単独で完結、移動・共有が容易
- 巨大画像を多数埋め込むとファイルサイズが増大する点は留意
